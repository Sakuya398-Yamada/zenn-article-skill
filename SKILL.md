# Zenn記事作成スキル

Zennプラットフォーム向けの技術記事・アイデア記事を作成するスキル。
Zenn独自のMarkdown記法とファイル構造に準拠した高品質な記事を生成する。

## 基本ワークフロー

### 1. 記事の要件確認

ユーザーに以下を確認する（明示されていない場合のみ質問する）:

- **テーマ・タイトル**: 何について書くか
- **記事タイプ**: `tech`（技術記事）or `idea`（アイデア記事）。デフォルトは`tech`
- **対象読者**: 初心者向け / 中級者向け / 上級者向け
- **トピックス**: 最大5つのタグ（例: `["react", "typescript"]`）
- **絵文字**: 記事カードに表示する絵文字1文字。ユーザーに確認せず、記事の内容に合った絵文字を自動で選択する

### 2. ファイル作成

#### ディレクトリ構成

```
articles/
└── <slug>.md    # slug: 12〜50文字、半角英小文字・数字・ハイフン・アンダースコアのみ
```

#### Frontmatter

```yaml
---
title: "[生成AI利用] 記事タイトル"
emoji: "📝"
type: "tech"
topics: ["topic1", "topic2"]
published: false
---
```

タイトルには必ず先頭に `[生成AI利用]` を付与する。ユーザーが指定したタイトルの前に付ける形式とする。

| フィールド | 必須 | 説明 |
|-----------|------|------|
| title | Yes | 記事のタイトル |
| emoji | Yes | カード表示用の絵文字1文字 |
| type | Yes | `tech` or `idea` |
| topics | Yes | タグ配列（最大5つ） |
| published | Yes | `true`で公開、`false`で下書き |
| published_at | No | 公開日時 `YYYY-MM-DD` or `YYYY-MM-DD hh:mm`（JST） |

slugの命名規則:
- 12〜50文字
- 使用可能文字: `a-z`, `0-9`, `-`, `_`
- 内容を端的に表す英語名にする（例: `setup-react-testing-library`）

### 3. 記事の構成

以下の構成を基本とする:

1. **導入部** - 何を解説するか、読者が得られるもの
2. **本文** - 見出し2（`##`）から始める。見出し1は使わない
3. **まとめ** - 記事の要点整理
4. **参考文献** - 記事作成にあたって参考にした記事・ドキュメントのリスト
5. **AI生成表記** - 記事末尾に必ず記載

記事末尾には以下の順で**リポジトリリンク**、**参考文献セクション**、**AI生成表記**を必ず記載する:

```markdown
## リポジトリ

この記事で扱っているソースコードは以下のリポジトリで公開しています。

https://github.com/ユーザー名/リポジトリ名

## 参考文献

- [記事タイトル1](URL1)
- [記事タイトル2](URL2)
- [ドキュメント名](URL3)

---

:::message
この記事は生成AI（Claude）を使用して作成されました。内容の正確性には注意を払っていますが、誤りがある可能性があります。お気づきの点があればコメントでお知らせください。
:::
```

**リポジトリセクション**について:
- 記事の内容に関連するソースコードのGitHubリポジトリURLを記載する
- リポジトリがまだ作成されていない場合は、セクションを残しつつ「リポジトリは準備中です」と記載する
- 記事にリポジトリが不要な場合（概念的な記事など）はセクション自体を省略してよい

**参考文献セクション**について:
- 記事作成にあたって参照した外部記事・公式ドキュメント・ブログ等をすべてリストアップする
- ユーザーから提示されたURLや、内容の根拠となった情報源を漏れなく記載する
- 参考文献がない場合でもセクション自体は残し、「なし」と記載する

**タイトルの `[生成AI利用]` プレフィックス**、**参考文献セクション**、**AI生成表記**は**すべての記事に必ず含める**こと。省略してはならない。

## Zenn Markdown記法リファレンス

記事を書く際は以下の記法を活用する。詳細は [references/zenn-markdown.md](references/zenn-markdown.md) を参照。

### 基本記法

- 見出し: `##` 〜 `####`（`#` 見出し1は使わない）
- 太字: `**テキスト**`
- イタリック: `*テキスト*`
- 打ち消し線: `~~テキスト~~`
- インラインコード: `` `code` ``
- リンク: `[テキスト](URL)`
- 画像: `![altテキスト](URL)`
  - 幅指定: `![](URL =250x)`
  - キャプション: 画像の次の行に `*キャプションテキスト*`
- 引用: `> テキスト`
- 区切り線: `-----`
- 脚注: 本文中に`[^1]`、末尾に`[^1]: 脚注内容`

### コードブロック

````markdown
```言語名
コード
```
````

- ファイル名付き: `` ```言語名:ファイル名 ``
- diff表示: `` ```diff 言語名 `` （行頭に `+` `-` `>` `<` ` ` が必要）

### Zenn独自記法

**メッセージボックス:**
```
:::message
通常のメッセージ
:::

:::message alert
警告メッセージ
:::
```

**アコーディオン（トグル）:**
```
:::details タイトル
折りたたみコンテンツ
:::
```

**数式（KaTeX）:**
- ブロック: `$$` で囲む（前後に空行が必要）
- インライン: `$数式$`

**コンテンツ埋め込み:**
- URLカード: URLだけの行、または `@[card](URL)`
- Twitter/X: ポストURLを貼るだけ
- YouTube: 動画URLを貼るだけ
- GitHub: ファイルURLを貼る（`#L1-L3`で行指定可）

### テーブル

```markdown
| ヘッダ1 | ヘッダ2 |
|---------|---------|
| セル1   | セル2   |
```

セル内改行は `<br>` を使用。

## 記事の品質ガイドライン

- 見出しは`##`から始める（アクセシビリティのため）
- コードブロックには必ず言語名を指定する
- 長いコードにはファイル名を付ける
- 重要な注意点は`:::message`や`:::message alert`で強調する
- 長い補足情報は`:::details`で折りたたむ
- 適切な箇所に画像やコード例を入れ、読みやすくする
- 専門用語には初出時に簡単な説明を添える

## 既存記事の編集

既存の`articles/`ディレクトリ内の記事を編集する場合:
1. まず対象ファイルを読み込む
2. frontmatterの形式を維持する
3. AI生成表記がない場合は追加する
4. ユーザーの指示に従って内容を修正する

## Zenn CLIとの連携

プロジェクトにZenn CLIがセットアップされている場合:
- `npx zenn new:article` で新規記事ファイルを生成可能
- `npx zenn preview` でローカルプレビュー（localhost:8000）
- GitHubリポジトリにpushすることでデプロイ

$ARGUMENTS が指定されている場合は、それを記事のテーマとして使用する。
